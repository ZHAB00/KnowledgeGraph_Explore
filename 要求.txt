技术实习生筛选项目：上线一个能用的小工具
你好，感谢投递。在约面试之前，我们想先给你一个贴近真实工作的小项目。
这不是算法笔试，也不考八股。我们招的是 AI native 的技术实习生（2027 届，表现好可转正），所以我们只看一件事：你能不能用 AI 和资料，把一个东西从 0 做到公网能访问，并讲清楚你是怎么做的。
●建议投入：核心部分 3–4 小时（用 AI 提效完全没问题，我们鼓励）
●提交期限：48 小时。从你在 BOSS 上回复「已收到，开始做」的那一刻起计时。如果这两天确有考试或其他硬冲突，请在开始前说明，可协商延长一次至 72 小时。
●AI 工具：随便用。ChatGPT / Claude / Cursor / Copilot / 任何 Agent 都可以。我们看结果和你的判断，不看你是不是「纯手写」。
第 0 部分：你的数字身份（5 分钟）
请直接列出以下链接（没有的项如实写「没有」，不扣印象分，但有会加分）：
1.GitHub 主页
2.个人博客 / 技术写作主页（博客、GitHub Pages、掘金、知乎专栏、Medium 均可）
3.X（Twitter）或其他你活跃的技术社区账号
4.你最拿得出手的一个作品链接（项目、网页、工具、开源贡献均可）
第 1 部分：核心项目（必做）
做一个单页小工具或产品页，并让它公网可访问。
三个方向任选其一（也可以自己提，附一句话说明为什么值得做）：
●方向 A：AI 模型价格计算器 —— 选 2–3 个海外 AI API（如 OpenAI、Anthropic、Replicate、fal.ai 等），做一个价格对比 + 用量估算页面。价格数据必须来自官方页面，注明来源和查询日期。
●方向 B：调用一个真实 API 的小工具 —— 任选一个有免费额度的公开 API（AI 类、数据类均可），做一个有实际输入输出的小工具页面。
●方向 C：你自己的想法 —— 任何你觉得 7 天内能被真人用上的小东西。
硬性要求（缺一项视为未完成）：
1.公网可访问：部署到 Vercel / Cloudflare Pages / Netlify 等，提交可打开的 URL。
2.GitHub 公开仓库：要有真实的 commit 历史（边做边提交），不要最后一次性传一个 commit。
3.英文 README：包含项目是什么、怎么本地跑起来、技术栈、已知限制（known limitations）。
4.移动端不崩：手机打开布局不乱、文字不溢出。
5.不要把任何 API key 提交到仓库里。
技术栈不限。React / Next.js / Vue / 原生 HTML 都行，选你最快能交付的。
第 2 部分：AI 协作记录（10 行以内）
●你用了哪些 AI 工具，分别用在哪个环节？
●哪一次 AI 给的结果是错的？你怎么发现、怎么修的？
●哪些代码或结论你到现在还不完全确定？
「AI 全程没出错」不是好答案——那通常说明你没验证。
第 3 部分：复盘博客（必做）
把这次项目写成一篇 500 字以上的复盘文章，公开发布，提交链接。
●发到你已有的博客最好；没有博客，就趁这次开一个（GitHub Pages / 掘金 / 知乎 / Medium 都行，10 分钟能搞定）。
●内容建议：做了什么、踩了什么坑、AI 在哪帮了你 / 坑了你、下次怎么做更快。
●写给「三个月前的自己」看，不用写成官方文档。
第 4 部分：5 行自评
1.这个项目你最满意的一点
2.最心虚的一点
3.再给你 8 小时，你会加什么 / 改什么
4.整个过程中你卡得最久的地方，卡了多久
5.如果入职第一周做类似任务，你希望我们提供什么

2.完成后，把你的提交文档整理成一个链接（Google Docs / 飞书 / Notion 均可，确保权限为「任何人可查看」），直接发在 BOSS 对话里，不用等我们催。
3.我们会在收到后 24 小时内回复你是否进入下一轮（30 分钟线上项目复盘）。
4.到期未提交视为放弃，我们不会另行通知。
提交文档里列清楚：

2.线上 URL + GitHub 仓库链接
3.AI 协作记录
4.博客文章链接

最后说明
●我们知道这道题有门槛：要能正常访问 GitHub、部署平台和海外 API 文档。这就是这份工作的日常环境，提前体验一下。
●不追求完美，追求真实、能跑、可复盘。一个简陋但真实上线的东西，远好于一份精美但跑不起来的 PPT。
●所有链接我们都会点开验证。查不到、打不开、对不上的内容，比「我没做到」糟糕得多。
